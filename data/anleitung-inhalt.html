Idiotensichere Anleitung: Dateien austauschen & auf GitHub hochladen

Diese Anleitung ist extra simpel geschrieben. Geh die Schritte genau der Reihe nach durch.

1) Vorbereitung (einmalig prüfen)

  1. Öffne das Projekt auf deinem Computer (Ordner HGH).
  2. Stelle sicher, dass du Internet hast.
  3. Stelle sicher, dass du auf GitHub eingeloggt bist.

2) Datei austauschen (Beispiel: Stundenplan oder Bild)

  1. Suche die alte Datei im Projektordner.
  2. Lege die neue Datei mit exakt gleichem Namen an die gleiche Stelle.
  3. Wenn gefragt wird: „Datei ersetzen?“ → auf Ja klicken.
Wichtig: Der Name und der Speicherort müssen stimmen. Sonst findet die Website die Datei nicht.

3) Änderungen auf GitHub hochladen (Web-Oberfläche)

  1. Öffne dein Repository auf GitHub.
  2. Klicke auf die Datei, die du ersetzt hast.
  3. Klicke oben rechts auf das Stift-Symbol oder auf Upload files.
  4. Ziehe die neue Datei in das Upload-Fenster.
  5. Unten bei der Nachricht kurz schreiben, was geändert wurde (z. B. Stundenplan KW 46 aktualisiert).
  6. Klicke auf Commit changes.

4) Prüfen, ob alles funktioniert

  1. Öffne die Website neu.
  2. Wenn du nichts Neues siehst: Seite hart neu laden (Windows: Strg + F5, Mac: Cmd + Shift + R).
  3. Prüfe genau die Stelle, an der du die Datei ersetzt hast.

5) Typische Fehler (und Lösung)

  • Falscher Dateiname: Datei exakt so nennen wie vorher.
  • Falscher Ordner: Datei in den gleichen Ordner wie die alte Datei legen.
  • Änderung nicht sichtbar: Browser-Cache mit harter Aktualisierung leeren.
